# v0.5.0 发布检查清单

## 当前状态

- `package.json` 版本：`0.5.0`
- `package-lock.json` 版本：`0.5.0`
- `src-tauri/tauri.conf.json` 版本：`0.5.0`
- `src-tauri/Cargo.toml` 版本：`0.5.0`
- `src-tauri/Cargo.lock` 应用包版本：`0.5.0`
- 目标 tag：`v0.5.0`

## 发布策略

`v0.4.1` 已经作为手工补丁 release 存在。为了避免继续回填旧 release，本次从包含 release workflow 的最新 `v0.4` 分支头部发布 `v0.5.0`。

## 发布前验证

```bash
npm ci --dry-run
npm test
npm run build
cd src-tauri
cargo check
```

## GitHub Actions 发布

推送 `v0.5.0` tag 后，`.github/workflows/release.yml` 会自动触发。

workflow 依赖仓库 Actions Secrets：

- `TAURI_SIGNING_PRIVATE_KEY`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

预期产物：

- Windows x64 NSIS 安装包
- updater 签名文件
- `latest.json`

## 失败排查

- 如果失败在依赖安装：确认 workflow 使用 Node 24，并检查 `package-lock.json` 与 `package.json` 是否同步。
- 如果失败在 action 解析：确认使用 `tauri-apps/tauri-action@v0`。
- 如果失败在签名：确认两个 Tauri signing secrets 均存在，且密码匹配私钥。
