## 发布前验证

```bash
npm test
npm run build
cd src-tauri
cargo check
```

建议额外做一次手动冒烟：

- 打开应用首页。
- 逐个点击侧栏 7 个入口。
- 打开播放器来源的 4 个 tab。
- 打开工具页，确认备份按钮与目录入口可见。

## GitHub Release 操作

发布时建议：

1. 确认 `v0.4` 分支已经推送。
2. 确认 `v0.4.0` tag 指向要发布的提交。如果 tag 仍指向旧提交，需要谨慎决定是否新建 `v0.4.1`，不要随意移动已发布 tag。
3. 创建 GitHub Release。
4. 上传安装包、签名文件和 `latest.json`。
5. 将 `RELEASE_NOTES_0.4.0.md` 内容作为 release body。

## 注意

当前 `v0.4.0` tag 创建早于后续 UI polish、README 修复和结构拆分。如果这些后续提交也要进入正式发布，推荐发布 `v0.4.1`，比移动既有 `v0.4.0` tag 更稳。
