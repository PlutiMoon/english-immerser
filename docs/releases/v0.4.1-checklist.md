`v0.4.0` tag 已经存在，且创建时间早于后续 UI polish、播放器结构拆分、README 修复和发布材料整理。为避免移动既有 tag，本次以 `v0.4.1` 发布最新提交。

## 构建产物检查

Windows NSIS 目标产物预计位于：

- `src-tauri/target/release/bundle/nsis/English Immerser_0.4.1_x64-setup.exe`

如果配置和签名环境齐备，还需要检查：

- `.sig` 签名文件
- `latest.json`

如果缺少 updater 元数据，需要检查 Tauri signing key 与 `createUpdaterArtifacts` 配置。

## 本地构建记录

- `npm test`：10 个测试文件、126 个测试通过。
- `npm run build`：通过。
- `cargo check`：通过。
- `npm run tauri build`：已生成 NSIS 安装包，但 updater 签名阶段失败。
- 当前环境未设置 `TAURI_SIGNING_PRIVATE_KEY`，因此没有生成可用于自动更新的 `.sig` 和 `latest.json`。
- 在补齐签名私钥前，GitHub Release 不应标记为 latest，避免自动更新入口指向缺少 updater 元数据的版本。

## GitHub Release 操作

1. 确认工作区干净。
2. 提交 v0.4.1 版本和文档改动。
3. 创建 `v0.4.1` tag。
4. 推送 `v0.4` 分支和 `v0.4.1` tag。
5. 使用 `RELEASE_NOTES_0.4.1.md` 创建 GitHub Release。
6. 上传安装包、签名文件和 `latest.json`。
